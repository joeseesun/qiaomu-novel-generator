## Complete Short Story Requirements

- Give the story a real ending. Do not stop at a trailer, synopsis, or chapter-one fragment unless the user asks for a serial opening.
- Default length: 1800-4000 Chinese characters.
- Put an irreversible disturbance within the first 3 paragraphs.
- Give the protagonist one visible desire and one hidden wound.
- Escalate conflict at least 3 times.
- Use concrete scenes. Avoid long explanations of background.
- Use dialogue to force choices, not to recite information.
- Use one recurring image or object and return to it near the end.
- End with resolution plus aftertaste: the core conflict should resolve, but the final image should echo beyond the page.

## Drafting Rules

- Prefer verbs and images over adjective stacks.
- Short sentences can create pressure; longer sentences can slow time. Use both intentionally.
- Do not explain the technique in the visible story.
- Do not put section labels inside the story unless the user asks for a structured artifact.
- If using a source fragment, keep facts and promises but rewrite scene logic when needed.
- If the user asks for a famous plot reference, extract only the underlying structure and create new names, setting, stakes, rules, and resolution.

## Revision Rules

If the story feels flat, diagnose the most important failure and fix that first:

1. Hook weak: start later, closer to the wound.
2. Desire weak: make the protagonist want something visible now.
3. Conflict weak: close the safe option.
4. Dialogue weak: turn information into accusation, bargain, or test.
5. Image weak: choose one object and make it carry memory or cost.
6. Ending weak: make the final image reframe the first scene.
